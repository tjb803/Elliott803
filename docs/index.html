Introduction

Project Aims

The aim of this project is to produce a simulation of the Elliott 803 computer that is capable of running original 803 programs (provided they can be made available as system files rather than real paper tapes). In particular it is an aim to run ALGOL 60 programs using the original Elliott ALGOL compiler and achieve correct results.

The project has been written in Java to enable it to run easily on most systems and the user interface elements have been written as simple Java Swing interfaces. Some elements of the interface such as the operator's console have been made to look and work in a manner very similar to the real 803 version (as far as the Swing interface allows); other elements such as the paper tape readers and punches are just simple implementations to allow easy operation and don't look anything like the real thing.

Notable Omissions

The following elements of the Elliott 803 are currently not implemented in this project.

  • Sounds
    The real 803 console contained a amplifier and loudspeaker that was pulsed by the CPU when executing certain instructions. This resulted in some very characteristic noises being produced during execution and by careful use of certain instructions and delays it was even possible to produce music.
    The simulation currently produces no sound at all, which some might say is a serious omission for an 803! The Java environment doesn't make creating this sort of sound particularly easy, but it is definitely something to be worked on in future versions.

  • Correct Instruction Timings
    No attempt has been made to emulate the precise timings of the machine instructions. The simulator simply runs as fast as the Java implementation allows (this is not especially fast, as the updates to the various Swing windows tend to slow it down a little).
    Again some might say this is a serious omission for an 803 emulation, but again it will probably be hard to get correct in a Java implementation and I don't feel it detracts too much from providing at least the "spirit" of the 803! (Note, to produce accurate sounds would also require accurate timings.)

  • Full Floating Point
    The simulation does fully implement all the group 6 floating point instructions but it currently does this using a little cheating. The Elliott 39-bit floating point values are converted to Java doubles and the operation is performed in Java, the resulting Java double is then converted back to an Elliott 39-bit float format.
    This is a simple and reasonably successful approach (ALGOL programs, for example, run correctly), but the resulting numbers are not always absolutely identical to those that would be produced by the real 803 hardware - sometimes there are minor differences in the least significant decimal places.

  • Missing Hardware
    Some bits of the original hardware are currently missing from the simulation. It is hoped to add these in the future:
    • Film Handlers (although most of the enjoyment of these was in seeing the real huge handlers working!).
